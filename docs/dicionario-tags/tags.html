Dicionário de TAGs — Sistema de Gerenciamento de IDs

Guia prático e didático para padronizar nomes de arquivos e pastas.

1) Visão geral

TAG = código curto no começo do nome do arquivo (e no id_item) que identifica o tipo do documento. Use nomes de pastas legíveis para humanos.

Formato geral de arquivo: TAG-AAAA-MM[ - Descritivo].ext (para documentos mensais).

O sufixo humano é opcional: adicione após “ - ” sem quebrar a validação.

2) Regras comuns

  • Arquivos de ICMS (DEVEC, LDO) ficam na pasta 11 ICMS/.
  • Pastas de Empresa/Unidade: “{SIGLA} - {id_empresa} / {NOME} - {id_unidade}” (ex.: CEOLIN - 0001 / Matriz - 001).
  • Notas de Energia (NE-CP, NE-LP, NE-CPC, NE-LPC, NE-VE): ficam na MESMA pasta “Notas de Energia/”.
  • CCEE: cada tipo/subcódigo (ex.: CFZ003) tem sua própria pasta dentro de “CCEE - DRI/”.

Exemplos de caminhos:

/dados/empresas/CEOLIN - 0001/Matriz - 001/
  Faturas/FAT-2025-01.pdf
  Notas de Energia/NE-CP-2025-02.pdf
  Notas de Energia/NE-LPC-2025-02.pdf
  11 ICMS/DEVEC-2025-06.pdf
  11 ICMS/LDO-2025-06.pdf
  BM Energia/DOC-CTR-2025.pdf
  BM Energia/MIN-CTR-2025 - Preço Fixo.docx
  CCEE - DRI/CFZ003/CCEE-CFZ003-2025-04.pdf

5) CCEE com subcódigo (cada tipo em uma pasta)

TAG do tipo: CCEE-<COD>, onde <COD> ∈ {CFZ, GFN, LFN, LFRCA, LFRES, PEN, SUM, BOLETOCA, ND}. Alguns possuem sufixo numérico (003, 004, 001 etc.), mas BOLETOCA e ND não utilizam sufixo.

Novos códigos incluídos: BOLETOCA, ND

Nome do arquivo: CCEE-<COD>-AAAA-MM[-Vn][-X][ - Descritivo].pdf

Exemplos e pastas:

CCEE - DRI/CFZ003/CCEE-CFZ003-2025-04.pdf

6) Regex de validação (opcional)

Use estas expressões para validar nomes (aceitam extensões comuns e, quando indicado, sufixo humano).

FAT  : ^FAT-\d{4}-(0[1-9]|1[0-2])\.(pdf|xlsx|csv)$
NE   : ^NE-(CP|LP|CPC|LPC|VE)-\d{4}-(0[1-9]|1[0-2])\.(pdf|xlsx|csv)$
DEVEC: ^DEVEC-\d{4}-(0[1-9]|1[0-2])\.(pdf|xlsx|csv)$
LDO  : ^LDO-\d{4}-(0[1-9]|1[0-2])\.(pdf|xlsx|csv)$
REL  : ^REL-\d{4}-(0[1-9]|1[0-2])\.(pdf|xlsx|csv)$
RES  : ^RES-\d{4}-(0[1-9]|1[0-2])\.(pdf|xlsx|csv)$
EST  : ^EST-\d{4}-(0[1-9]|1[0-2])\.(pdf|xlsx|csv)$
DOC  : ^DOC-[A-Z]{3}-.*\.(pdf|docx|xlsx)$
MIN  : ^MIN-[A-Z]{3}-.*\.(pdf|docx|xlsx)$
CCEE : ^CCEE-(?:CFZ\d{3}|GFN\d{3}|LFN\d{3}|LFRCA\d{3}|LFRES\d{3}|PEN\d{3}|SUM\d{3}|BOLETOCA|ND)-\d{4}-(0[1-9]|1[0-2])(-V\d+)?(-[A-Z])?( - .+)?\.(pdf|xlsx|csv)$

8) Boas práticas

  • Arquivos sempre começam pela TAG.
  • Pastas de Empresa/Unidade: “Nome - ID”.
  • NE-CP, NE-LP, NE-CPC, NE-LPC e NE-VE juntos em “Notas de Energia/”.
  • CCEE: uma pasta por tipo/subcódigo (ex.: CFZ003/).
  • Para DOC/MIN jurídico, use data completa no ID.
  • Revisões: use -Vn; múltiplos no mês: -NNN.
  • Sufixo humano opcional: “ - Descritivo” no fim do nome.
